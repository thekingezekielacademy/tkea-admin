# üìö Batch Class System - Complete Documentation

## üéØ System Overview

The TKEA Batch Class System operates as a **school-like learning platform** where 5 different classes run on different days of the week, each progressing day-by-day through their curriculum. **Each class starts on its designated weekday**, and **new batches start every week** on that same day, allowing multiple batches of the same class to run concurrently.

---

## üìñ The 5 Classes & Their Start Days

Each class has its **own designated start day** and progresses sequentially:

1. **FREELANCING - THE UNTAPPED MARKET** ‚Üí Starts **MONDAY**
2. **INFORMATION MARKETING: THE INFINITE CASH LOOP** ‚Üí Starts **TUESDAY**
3. **YOUTUBE MONETIZATION: From Setup To Monetization** ‚Üí Starts **WEDNESDAY**
4. **EARN 500K SIDE INCOME SELLING EBOOKS** ‚Üí Starts **THURSDAY**
5. **CPA MARKETING BLUEPRINT: TKEA RESELLERS - TOTALLY FREE** ‚Üí Starts **FRIDAY**

## üóìÔ∏è How The System Works

### Class Progression Pattern

  - Monday = Class 1

  - Friday = Class 4

### Weekly Batch Creation

**New batches start every week on each class's designated day:**

- **Every Monday:** New FREELANCING batch starts (Class 1)

- **Every Wednesday:** New YOUTUBE batch starts (Class 1)

**Multiple batches of the same class run simultaneously:**

- Batch 1 might be on Class 6

- Batch 3 might be on Class 4
- Batch 4 might be on Class 3
- Batch 5 might be on Class 2
- Batch 6 (NEW) starts with Class 1

**All batches progress day-by-day independently.**

## üîî Notification System

### Notification Schedule

Notifications are sent at **5 specific intervals** before each class session starts:

1. **5 days** before class start
2. **48 hours** before class start
3. **24 hours** before class start
4. **3 hours** before class start
5. **30 minutes** before class start

### Notification Types

#### üìÖ Enrolment Update Notification
**Template:**
```
üìÖ Enrolment Update: A new session for [Name of Course] is officially scheduled for [Date] at [Time]. Perfect for all new members!
```

**Variables:**
- `[Name of Course]` - One of the 5 class names
- `[Date]` - The scheduled date (e.g., "Monday, January 19th")
- `[Time]` - The scheduled time (e.g., "6:30 AM", "1:00 PM", or "7:30 PM")

### Notification Triggers

Notifications are automatically triggered based on:
- **Class start day** (Monday for FREELANCING, Tuesday for INFORMATION MARKETING, etc.)
- **Batch start date** (when the batch was created)
- **Session number** (Class 1, Class 2, etc.)
- **Session time** (morning 6:30 AM, afternoon 1:00 PM, evening 7:30 PM)

## üèóÔ∏è Technical Implementation Requirements

### Database Schema Needs

1. **Batches Table**
   ```sql
   CREATE TABLE batches (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     class_name TEXT NOT NULL, -- One of the 5 class names
     batch_number INTEGER NOT NULL, -- Sequential per class (1, 2, 3...)
     start_date DATE NOT NULL, -- The Monday/Tuesday/etc when batch started
     start_day_of_week INTEGER NOT NULL, -- 0=Monday, 1=Tuesday, etc.
     status TEXT DEFAULT 'active' CHECK (status IN ('active', 'completed')),
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
     updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
     UNIQUE(class_name, batch_number)
   );
   ```

2. **Class Sessions Table** (Modified from existing)
   ```sql
   CREATE TABLE class_sessions (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     batch_id UUID NOT NULL REFERENCES batches(id) ON DELETE CASCADE,
     class_name TEXT NOT NULL, -- One of the 5 class names
     session_number INTEGER NOT NULL, -- Class 1, Class 2, etc.
     session_title TEXT NOT NULL, -- Video name/title
     course_video_id UUID REFERENCES course_videos(id),
     session_type TEXT NOT NULL CHECK (session_type IN ('morning', 'afternoon', 'evening')),
     scheduled_date DATE NOT NULL, -- The actual date (Monday, Tuesday, etc.)
     scheduled_time TIME NOT NULL, -- 06:30:00, 13:00:00, 19:30:00
     scheduled_datetime TIMESTAMP WITH TIME ZONE NOT NULL,
     status TEXT DEFAULT 'scheduled' CHECK (status IN ('scheduled', 'in_progress', 'completed', 'cancelled')),
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
     updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
   );
   ```

3. **Classes Configuration Table**
   ```sql
   CREATE TABLE batch_classes (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     class_name TEXT NOT NULL UNIQUE, -- One of the 5 class names
     course_id UUID REFERENCES courses(id), -- Link to course if exists
     start_day_of_week INTEGER NOT NULL, -- 0=Monday, 1=Tuesday, etc.
     total_sessions INTEGER NOT NULL, -- How many classes in curriculum
     is_active BOOLEAN DEFAULT true,
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
     updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
   );
   ```

4. **User Batch Enrollment Table**
   ```sql
   CREATE TABLE user_batch_enrollments (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
     batch_id UUID NOT NULL REFERENCES batches(id) ON DELETE CASCADE,
     class_name TEXT NOT NULL, -- One of the 5 class names
     enrollment_date DATE NOT NULL DEFAULT CURRENT_DATE,
     access_level TEXT NOT NULL DEFAULT 'limited_access' CHECK (access_level IN ('full_access', 'limited_access')),
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
     UNIQUE(user_id, batch_id)
   );
   ```

5. **Notifications Table** (Modified from existing)
   ```sql
   CREATE TABLE batch_class_notifications (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
     session_id UUID NOT NULL REFERENCES class_sessions(id) ON DELETE CASCADE,
     notification_type TEXT NOT NULL CHECK (notification_type IN ('5_days', '48_hours', '24_hours', '3_hours', '30_minutes')),
     scheduled_send_time TIMESTAMP WITH TIME ZONE NOT NULL,
     sent_at TIMESTAMP WITH TIME ZONE,
     status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'sent', 'failed')),
     error_message TEXT,
     created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
     UNIQUE(session_id, user_id, notification_type)
   );
   ```

### Automated Processes

1. **Weekly Batch Creation (Per Class)**
   - **FREELANCING:** Cron job runs every Monday at 00:00 UTC
   - **INFORMATION MARKETING:** Cron job runs every Tuesday at 00:00 UTC
   - **YOUTUBE:** Cron job runs every Wednesday at 00:00 UTC
   - **EBOOKS:** Cron job runs every Thursday at 00:00 UTC
   - **CPA MARKETING:** Cron job runs every Friday at 00:00 UTC
   
   **Action:**
   - Get last batch number for that class
   - Create new batch with incremented batch number
   - Set start_date to current day (Monday/Tuesday/etc.)
   - Auto-enroll any pending users

2. **Daily Session Generation**
   - Runs daily at 00:00 UTC
   - For each active batch:
     - Calculate current session number based on days since batch start
     - Account for class's start day of week
     - Create 3 sessions (morning, afternoon, evening) for that day
     - Assign video based on session_number (sequential, not cyclical)

3. **Notification Scheduling**
   - Trigger: When a new session is created
   - Action:
     - Calculate 5 notification times (5 days, 48h, 24h, 3h, 30m before)
     - Create notification records for all enrolled users in that batch
     - Queue notifications for background processing

4. **Notification Sending**
   - Trigger: Background job runs every minute
   - Action:
     - Check for notifications where `scheduled_send_time <= now()` AND `status = 'pending'`
     - Send notification (email, Telegram, etc.)
     - Update `sent_at` and `status = 'sent'`

5. **Access Control Check**
   - Middleware/function checks user's access level
   - Validates if user can replay based on enrollment and access_level
   - Returns appropriate response (content or popup trigger)

### API Endpoints Needed

1. **GET /api/batches/current** - Get current active batches for all classes
2. **GET /api/batches/:className/current** - Get current batches for specific class
3. **GET /api/batches/:batchId/sessions** - Get all sessions for a batch
4. **GET /api/sessions/:sessionId** - Get specific session details
5. **POST /api/batches/enroll** - Enroll user in a batch
6. **GET /api/users/:userId/batches** - Get user's enrolled batches
7. **GET /api/sessions/:sessionId/replay** - Check replay access and return content or popup
8. **POST /api/notifications/schedule** - Schedule notifications for a session

### Frontend Components Needed

1. **BatchSelector** - Component to show available batches per class and allow enrollment
2. **ClassSessionCard** - Display individual class session with date, time, title
3. **ReplayButton** - Button that checks access and shows popup if needed
4. **AccessRestrictedPopup** - Popup component with bank transfer details and upgrade button
5. **NotificationSettings** - User preferences for notifications
6. **BatchProgressTracker** - Show user's progress through their batch
7. **ClassScheduleView** - Show all 5 classes and their current sessions

## üîÑ Automation Rules

### Batch Creation (Per Class)

**FREELANCING:**
- **Trigger:** Every Monday at 00:00 UTC
- **Action:**
  - Get last batch number for FREELANCING
  - Create new batch (batch_number = last + 1)
  - Set start_date to current Monday
  - Set start_day_of_week = 0 (Monday)
  - Auto-enroll any pending users

**INFORMATION MARKETING:**
- **Trigger:** Every Tuesday at 00:00 UTC
- **Action:** Same as above, but start_day_of_week = 1 (Tuesday)

**YOUTUBE:**
- **Trigger:** Every Wednesday at 00:00 UTC
- **Action:** Same as above, but start_day_of_week = 2 (Wednesday)

**EBOOKS:**
- **Trigger:** Every Thursday at 00:00 UTC
- **Action:** Same as above, but start_day_of_week = 3 (Thursday)

**CPA MARKETING:**
- **Trigger:** Every Friday at 00:00 UTC
- **Action:** Same as above, but start_day_of_week = 4 (Friday)

### Session Generation

- **Trigger:** Daily at 00:00 UTC
- **Action:**
  - For each active batch:
    - Calculate days since batch start_date
    - Account for class's start_day_of_week
    - Calculate current session_number
    - Get today's date and determine if it's a class day
    - If yes, create 3 sessions (morning, afternoon, evening)
    - Assign video based on session_number (sequential)

**Example Calculation:**
- FREELANCING Batch 1 started Monday, Jan 19th
- Today is Wednesday, Jan 21st
- Days since start: 2 days
- Session number: 3 (Class 3)
- Create 3 sessions for Wednesday with session_number = 3

### Notification Scheduling

- **Trigger:** When a new session is created
- **Action:**
  - Calculate 5 notification times:
    - 5 days before: `scheduled_datetime - 5 days`
    - 48 hours before: `scheduled_datetime - 48 hours`
    - 24 hours before: `scheduled_datetime - 24 hours`
    - 3 hours before: `scheduled_datetime - 3 hours`
    - 30 minutes before: `scheduled_datetime - 30 minutes`
  - Create notification records for all enrolled users in that batch
  - Queue notifications for background processing

### Notification Sending

- **Trigger:** Background job runs every minute
- **Action:**
  - Check for notifications where `scheduled_send_time <= now()` AND `status = 'pending'`
  - Send notification (email, Telegram, etc.)
  - Update `sent_at` and `status = 'sent'`
  - If failed, update `status = 'failed'` and log `error_message`
